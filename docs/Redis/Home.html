    # Redisについて

    ここはRedisのことを書いてるよ

    # redisの環境構築

    # Dockerでredisの環境を作成する

    次のdocker-composer.ymlですぐにredisを立ち上げられるようにする

    version: "3"
services:
  redis:
    image: redis:latest
    container_name: redis
    ports:
      - "6379:6379"
    command: redis-server --appendonly yes
    volumes:
       - ./data/redis:/data
    hostname: redis-server

    # Redis server の 起動方法

    次のコマンドでサーバを起動する

    $ redis-server --appendonly yes

    appendonly yesは、障害とかが発生してデータが消えることを防ぐためのAOFをオンにする設定

    終了するならprocessを終わらせればいいんじゃないかな

    # AOF（Append Only File）

    RDBファイルで自動保存するだけでも問題ないけど、障害とかが発生した時にデータを保存されないことを防ぐための機能

    デフォルトだと1秒に1回fsyncして記録する

    AOFは、実行された書き込みコマンドを記録してデータを保存するようにしてる

    Redis AOF は、MySQL や MongoDB みたいに、現在の状態をインクリメンタルにアップデートするように動くよ

    # Redis client の 起動方法

    Redisのデータを直接操作するための対話式プログラムが起動するよ

    $ redis-cli

    # Redis client を終了する方法

    Redisのclientの対話式プログラム内で次のコマンドを実行すればclientは終わる

    > exit

    # Redis で RDBファイルを作って終了する

    Redisのclientの対話式プログラム内で次のコマンドを実行すればRDBファイルが作られてclientとserverが終わる

    > shutdown

    # データベースの切り替え

    Redisのデータベースは、0から始まる整数値のindexを持っているからそれで指定するよ

    切り替える時は、selectコマンドで切り替え

    > select [index]

    exist コマンドを使うとキーに値がすでにあるか確認できるよ
