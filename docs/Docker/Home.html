    # Docker Composeのデータの永続化の方法

    Docker-composeを使った時にDBのデータとかでデータを永続化させる時に使える方法を紹介

    # Version 2の場合

    2つ方法があってサービスにvolumesをつける方法とbusyboxを使う方法があるよ

    # よく使われる方法

    サービスに直接volumesを設定する方法

    データは、valumesに書いたディレクトリにデータが作られるよ

    version: "2"
services:
  database:
    image: mariadb
    volumes:
      - ./docker/mariadb/data:/var/lib/mysql

    # busyboxを使う方法

    Version2の時は、busyboxを使うことが勧められていたっぽい

    version: "2"
services:

  storage:
    image: busybox
    container_name: storage
    volumes:
      - ./docker/mariadb/data:/var/lib/mysql

  database:
    image: mariadb
    container_name: database
    environment:
      MYSQL_ROOT_PASSWORD: password
    ports:
      - 3306
    volumes_from:
      - storage

    # Version 3の場合

    serviceと同じインデント？階層にvolumesを設定するとできる

    version: "3"
services:

  database:
    image: mariadb
    container_name: database
    environment:
      MYSQL_ROOT_PASSWORD: password
    ports:
      - 3306
    volumes:
      - storage:/var/lib/mysql
volumes:
	- storage:

    この方法だとデータが上二つと違って出力される場所が指定できない。

    # Docker hubにDockerfileをpushする方法

    # 1.Dockerfileを作る

    テキトーにpushしたいオリジナルのdockerfileを作る

    $ vim Dockerfile

    # 2.tagをつけてbuildする

    Dockerfileがあるディレクトリでユーザ名とコンテナ名をtagにしてbuildする

    $ docker build --tag=[ユーザ名]/[コンテナ名] ./

    # 3.pushする

    # 3-0（オプション）ログインする

    CLIからdockerにログインし忘れていたらログインしておくこと

    $ docker login

    # 3-1.pushする

    このコマンドだけでpush完了！

    $docker push [ユーザ名]/[コンテナ名]

    # Dockerのvolumeの中身の行方

    # Dockerのvolumeのリスト

    Dockerで作ったvolumeに何があるか調べる時は下のコマンドを実行する

    $ docker volume ls

    下のような感じで出力される

    DRIVER              VOLUME NAME
local               weblog_logs

    # DockerのVolumeの情報

    ls のVOLUME NAMEを使って、下のコマンドでVolumeのことを調べられるよ

    $ docker volume inspect {VOLUME NAME}

    結果はこんな感じ

    [
    {
        "CreatedAt": "2019-08-19T15:18:23Z",
        "Driver": "local",
        "Labels": {
            "com.docker.compose.project": "weblog",
            "com.docker.compose.version": "1.24.1",
            "com.docker.compose.volume": "logs"
        },
        "Mountpoint": "/var/lib/docker/volumes/weblog_logs/_data",
        "Name": "weblog_logs",
        "Options": null,
        "Scope": "local"
    }
]

    # Volumeの削除

    ls のVOLUME NAMEを使って、下のコマンドでVolumeを削除できるよ

    $ docker volume rm {VOLUME NAME}

    # Volumeの全削除

    rmだと指定したものを削除する設定だったけど、下のpruneでVolumeを削除できるよ

    $ docker volume prune
