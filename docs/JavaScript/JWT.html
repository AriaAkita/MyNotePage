    # JWTについて

    JWT

    # JWT(JSON Web Token)

    JSON Web Tokenの略で、クレーム属性情報を安全にやり取りするためのJSONデータ構造で表したTokenの仕様のこと

    クレームのまとめたもJSONオブジェクト文字列をJWSやJWEにエンコードして、クレームのデジタル署名やMACと暗号化の両方ができる

    TIP

    読み方はjotで同じ発音

    # JWTの構造

    JWTは基本JSONで次のデータ構造を持ってる

    実際のデータはBase64urlでエンコードされていて、keyとvalueのペアになるようになってる

    • JWT ヘッド
      • パラメータ名
      • パラメータ値
    • JWTクレームセット（ペイロード）
      • クレーム
        • クレーム名
        • クレーム値
    • 署名

    # JWTヘッダ

    アルゴリズムとTokenのタイプを設定するよ

    {
	"alg": "HS256",
	"typ":"JWT"
}

    # JWTクレーム

    JWTクレームセットはJSONオブジェクトで、それぞれのメンバはJTWとして送られるクレームになる

    ちなみにクレーム名は一意でないといけない

    JWTを受信する時に、クレーム名が重複したJWTは拒否するか重複している最後のメンバ名だけ編訳するJASONパーサを利用しなければいけない

    # 予約済クレーム名

    下にあげるクレーム名は予約済みのクレーム名なので使用できないよ

    # "iss"（Issuer）クレーム

    iss(Issuer)クレームはJWTの発行者の識別子のこと

    # "sub"（Subject）クレーム

    sub（subject）クレームはJWTの主語となる主体

    # "aud"（Audience）クレーム

    aud（Audience）クレームはJWT

    # "exp"（Expiration Time）クレーム

    exp

    # "nbf" (not before) クレーム

    nbf

    # "iat" (Issued At) クレーム

    iat

    # "jti" (JWT ID) クレーム

    Jti

    # "typ" (Type) クレーム

    typ

    # JWS(JSON Web Signeture)

    JSONが元のデータ構造で、デジタル証明やMessage Authentication Codes(MACs)で保護されたコンテンツを表現するためのもの

    JWSヘッダとJWSペイロード、JWS 署名 で構成される
