    # IIFE（即時実行関数式）

    IIFE (Immediately Invoked Function Expression; 即時実行関数式) は定義されると、すぐに実行される JavaScript の 関数 のこと

    (function () {
    文
})();

    最初の()は、グループ化演算子に囲まれた静的スコープ

    最後の()は、関数の()のことでこれがあるので直接関数を実行するよと解釈する

    # ECMAScript2015からの変数の宣言

    # 定数のconst

    constは、再宣言と再代入ができない変数の宣言だよ

    # 再代入ができない

    一度初期化をしたconstの変数に代入はできないよ

    const NAME = "Akita"

try {
    // 再代入できないからエラーになるよ
	NAME = "Blue"
 } catch (e) {
    console.log(e)
}

    # 値の変更

    constの値は不変じゃないよ

    再代入はできないけど、値を変えることとはできる

    const NAMES = []

// 配列や連想配列の中の追加や削除はできるよ
NAMES.push('Akita')
NAMES.push('Blue')

    # 再宣言ができない

    再宣言は、同じスコープ内ではすることができないよ

    try {
    const AGE = 9
    // 再宣言は同じスコープ内だとできないよ
    const AGE = 12
} catch (e) {
    console.log(e)
}

    # Windowオブジェクトにならない

    varで宣言した変数と違って windowオブジェクトのプロパティにはならないよ

    const YEAR = 0

// Undefindになるよ
cnosole.log(window.YEAR)

    # ECMAScript2015からの変数の宣言

    # 再宣言ができないlet

    letを使うと同じスコープ内で、同じ名前の変数や関数を再宣言できない変数の宣言ができるよ

    let age = 2

try {
	let age = 9
} catch (e) {
	console.log(e)
}

    # Windowオブジェクトにならない

    varで宣言した変数と違って windowオブジェクトのプロパティにはならないよ

    let today = '2020/05/24' 
 
// Undefindになる
console.log(window.today)

    # ECMAScript2015からの Arrow Function とは

    # Arrow Functionってなに？

    無名関数のfunctionの省略記法のことだよ

    (args) => { ..処理..}みたいに書くよ

    argsが一つの場合は()を省略できるよ

    const func = (arg,...,arg) => {
    // 処理
}

    # 二つの文体

    Arrow Functionだと、処理のところConcise BodyとBlock Bodyのどちらかを使えるよ

    • Concise Bodyは、単一の式のみを書くことができて、その式がreturnされる値になるよ
    • Block Bodyは、functionと同じ書き方ができるよ
    //Consise Body
const conciseBodyFunction = arg => arg + arg;

// ブロック体
var func = (arg) => {
    return arg + arg;
};
