    # Vue.js

    # vue jsのbeta版を入れる方法

    $ yarn global add @vue/cli@next

    # VuePressの入れ方

    Vue.js 、VuePress

    せっかくノートを書いてるから、VuePressを導入してノートを外からでも見れるようにしてみた。

    Nuxt.jsとかもあるけど、静的なサイトを作るのにいいけど、Documentを作成するならVuePressの方が楽そうだった

    # 0.いつもの準備

    いつも通りにyarnでプロジェクトを作成する

    いつも書いてるから説明はなし！

    # 1.VuePressをプロジェクトにインストール！

    次のコマンドでインストールする

    $ yarn add vuepress

    # 2.出力先ディレクトリを作成

    docsディレクトリを作って、ここにVuePress用のファイルを出力する場所にする

    $ mkdir docs

    TIP

    GitPageを使う場合に、docs ディレクトリをルートディレクトリにする設定があるからdocsに出力するようにしたよ

    # 3.config.jsファイルで設定

    VuePressの 設定はここの .vuepress/config.jsに記載する

    ここの設定は、ほぼこんな感じ

    module.exports = {
    title: 'Aria Akita\'s Notes',
    description: 'ここでは、Aria Akitaが仕事や自分で勉強したことをまとめています',
    locales: {
        '/': {
            lang: 'ja'
        }
    },
    themeConfig: {
        nav: [
            {text: 'Home', link: '/'}
        ],
        sidebar: [
            '/',
        ],
        displayAllHeaders: true,
        smoothScroll: true
    },
    head: [
        ['meta' ,{ charset: 'utf-8' }],
        ['meta' ,{ name: 'viewport', content: 'width=device-width, initial-scale=1' }],
        ['meta' ,{ name:'robots', content:'noindex' }]
    ],
    base: '/MyNotePage/',
    dest: 'docs',
}
    • baseは、GitPage用にhttps://ariaakita.github.io/MyNotePage/の/MyNotePage/をBaseとするためにつけるよ
    • destは、出力先のディレクトリを指定するよ

    # 4.package.jsonのscriptsに設定を追加

    開発用とビルド用Scriptの実行コマンドを追加するなら下のような感じになるよ

    "scripts": {
    "dev": "vuepress dev",
    "build": "vuepress build"
  },

    デバックするときは、yarn run devを使って、本番用にデータを生成するならyarn run buildをするようにしたよ
