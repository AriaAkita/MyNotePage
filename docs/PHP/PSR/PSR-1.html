    # PSR-1

    2019/08/25

    # 1.概要

    • ファイルでは、PHPタグは↓のように書くこと
    <? php 

    <?=
    • PHPファイルの文字コードは、UTF-8 without BOMだけを使用すること
    • ファイルには、シンボル（クラス、関数、定数など...)の宣言と副作用の原因の処理（出力の生成、ini設定を変更など...）を書く必要があるけど、1つのファイルに両方を書かないこと
    • 名前空間とクラスはauto-lodingに必ず合わせること
      • PSR-0 と PSR-4を見ると書いてあるよ。
    • クラス名は、StudlyCaps記法で必ず書くこと
      • Ex. ClassName、AdminClass
    • クラスで定数を宣言するときは、すべて大文字にしてスペースは_で必ず書くこと
      • Ex.CONST_VALUE, WORD_LENGTH
    • メソッド名はcamelCaseで記述すること
      • Ex.currentTime, valueLength

    # 2.ファイル

    # 2.1.PHP Tags

    長いPHPのコードを書くときは<?php ... ?>を使うこと

    短いechoをするコードでは、<?= ... ?>を使うこと

    # 2.2.文字エンコーディング

    PHPのコードは、UTF-8のBOMなしを使うこと

    # 2.3 副作用

    ファイルに、新しいシンボル（クラスや関数、定数など...)と副作用のある処理は一緒に書かないこと

    ここでいうところの副作用とは、クラスや関数、定数の宣言と関係ないロジックのこと

    （後で書く）

    # 3.名前空間とクラス名

    名前空間とクラスはautoloadingに必ず合わせること

    PSR:PSR-0 と PSR-4に書いてあるよ。

    各クラスには、少なくとも１つレベル（トップレベルのベンダー名）の名前空間を持つことになるよ。

    クラス名は必ずStudlyCapsで名前をつけること

    PHP5.3より上のバージョンだと、正しい名前空間を使うこと

    （正直7.0より下はサポートが切れてるから、5.2以下のことは考えないよ）

    Ex.

    <?php
// PHP 5.3 and later:
namespace Vendor\Model;

class Foo
{
  
}

    # 4.クラスや定数、プロパティ、メソッドについて

    ここでいうところのクラスは、全てのクラス と インタフェース、特性のこと

    • 特性ってなに？

    # 4.1.定数

    クラスの定数は必ず upper case と underscore で区切って書くこと

    Ex.

    <?php
namespace Vendor\Model;

class Foo
{
    const VERSION = '1.0';
    const DATE_APPROVED = '2012-06-01';
}

    # 4.2.プロパティ

    この規約で、プロパティの命名規則に特定の書き方（$S¥¥$StudlyCaps、$camelCase、$under_scoreとか）は推奨することはないよ。

    ただ、どんな命名をするにしても適切なスコープ内で、一貫させること

    （ここでのスコープは、ベンダーレベルやパッケージレベル、クラスレベル、メソッドレベルのこと）

    # 4.3.メソッド

    メソッド名は camelCase()で書くこと
