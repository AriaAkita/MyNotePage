    # macOSでcomposer install した時にphp-intlでエラーになった時

    # 始まり

    composer installをしたら、php-intlがないと怒られてできなかった....

    調べてみたら、MacOSのデフォルトのPHPでは 、php-intlが入っていなかったのが原因っぽい

    # おまけ

    homebrewもhomebrew/phpがdeprecatedになり、homebrew/coreに含まれたので同じことが起きているっぽい

    # 1.新しくPHP@7.3を入れる

    このコマンドを入力して新しいPHPのバージョンを入れる！

     $ brew install php@7.3

    # 2.パスを通す

    パスを通して使えるようにする

    $ echo export PATH="/usr/local/opt/php@7.1/bin:$PATH" >> ~/.zshrc
$ echo export PATH="/usr/local/opt/php@7.1/sbin:$PATH" >> ~/.zshrc

    一応確認をしておく

    $ which php

    # 3.古い設定ファイルを消す

    今使っている設定ファイルだとphp-intlが使えないので、次のフォルダのconf.dをどっかにおく

    $ mv /usr/local/etc/php/{バージョン}/conf.d /usr/local/etc/php/{バージョン}/conf.d.bk

    これでOK！
